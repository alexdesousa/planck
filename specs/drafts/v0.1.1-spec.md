## Phase 3 ŌĆö Local node tools

Allow apps depending on `planck_headless` to register tools that run in the
same BEAM node without a sidecar.

**API:**
- `Planck.Headless.register_tool(tool)` ŌĆö adds a `Tool.t()` to `ResourceStore`
  globally; available to all new sessions
- `Planck.Headless.start_session(tools: [tool])` ŌĆö per-session extra tools
- Both feed into the same `tool_pool` that agents resolve against via TEAM.json
- `execute_fn` closures run in the host node with no RPC overhead

**Spec:** `specs/local-tools.md` (to be written)

## Phase 4 ŌĆö `checkpoint_agent` tool

Orchestrator tool to insert a `{:custom, :summary}` checkpoint into a worker's
conversation. Messages before the checkpoint are archived in the DB (visible
to the user) but invisible to the LLM on subsequent turns.

**API:**
- `Agent.checkpoint(agent, summary_text)` ŌĆö new public function
- `checkpoint_agent` tool added to `orchestrator_tools`
- Parameters: `identifier`, `identifier_type`, `summary` (required)
- Safe against self-targeting

**Spec:** `specs/checkpoint-agent.md`

## Phase 5 ŌĆö Dynamic skill injection + hot reload

Decouple skill descriptions from `state.system_prompt` so running agents pick
up skill changes without restarting.

**Changes:**
- `AgentSpec` stores `skill_names: [String.t()]` instead of baking skill
  descriptions into `system_prompt`
- `do_run_llm` resolves skill descriptions fresh from `ResourceStore` per turn
- New `Planck.Headless.Watcher` GenServer ŌĆö watches `.planck/skills/`,
  `.planck/teams/`, `config.json`, `.planck/.env`; debounced 300ms;
  calls `ResourceStore.reload/0` on changes
- `file_system` added to `planck_headless` deps

**What does NOT hot-reload:** agent identity line, user-written system prompt
from TEAM.json, built-in tool closures.

**Spec:** `specs/hot-reload.md`
