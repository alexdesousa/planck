# planck_docker

A Docker Compose stack that runs Planck with private web search, workspace
indexing, and document extraction. Designed to be installed once by a technical
user for themselves or for someone else. The same setup covers both local-only
access (default) and network-accessible server deployments.

## Use cases

- **Local workstation** — install on your own machine; access at `localhost:4000`
- **Someone else's machine** — install for a non-technical user; they open a browser, nothing else to configure
- **Home server** — expose on the local network so other household members can access it from their own devices

## Stack

```
Docker Compose
├── planck          (Elixir web UI + agent runtime + sidecar, Node.js included)
└── sidecar         (tools: web search via Searxng, document parsing via Tika, full-text search via Typesense)
```

All services communicate on an internal Docker network. Only `planck:4000`
is optionally exposed on the host — all other services are internal. All
containers run as UID/GID 1000 so files written to the workspace are owned
by the user.

## Network access

Controlled by `PLANCK_BIND_ADDRESS` in `$HOME/planck/.env`:

| Value | Use |
|---|---|
| `127.0.0.1` (default) | Local only — no ports open to the network |
| `0.0.0.0` | All interfaces — accessible from other devices |

```yaml
# compose.yml
ports:
  - "${PLANCK_BIND_ADDRESS:-127.0.0.1}:4000:4000"
```

## Directory layout

```
$HOME/planck/
  .env                        ← generated on first install; edit to configure
  typesense-data/             ← Typesense index (bind mount, survives docker compose down)
  workspace/                  ← mounted into the planck container as cwd
    .planck/
      config.json             ← written on first install only; preserved afterwards
      sidecar/                ← written on first install only; preserved afterwards
      sessions/
      skills/                 ← user-managed
      teams/                  ← user-managed
```

`~/.planck/` (the global Planck CLI config) is never touched.
The planck container's cwd is `$HOME/planck/workspace/`, so it picks up
`.planck/config.json` from there and never reads from `~/.planck/`.

## `.env` file

Generated by `install_docker.sh` on first install using an `add_if_missing`
pattern — existing keys are never overwritten:

```bash
# $HOME/planck/.env
TYPESENSE_API_KEY=planck-internal-key
PLANCK_BIND_ADDRESS=127.0.0.1
SEARXNG_SECRET=<random-secret-generated-by-install-script>
```

Edit `PLANCK_BIND_ADDRESS` to `0.0.0.0` to expose on the network.
Docker Compose reads this file automatically.

## Internal service credentials

Typesense requires an API key. Since it is internal to the Docker network
and never bound to the host, a fixed key in `.env` is safe:

```yaml
services:
  typesense:
    environment:
      TYPESENSE_API_KEY: ${TYPESENSE_API_KEY}
  planck:
    environment:
      TYPESENSE_API_KEY: ${TYPESENSE_API_KEY}
```

## Searxng configuration

A `settings.yml.template` is bundled in the planck image. On first run the
entrypoint renders it via `envsubst` (substituting `SEARXNG_SECRET`) and writes
it to `.planck/searxng/settings.yml` in the workspace. The Searxng container
mounts that file, so the user can edit it directly in the workspace.

Users can customise Searxng through Planck via the `configure-searxng` skill.
The skill points agents to `.planck/searxng/settings.yml` and documents the key
options; agents use `read`/`edit` to apply changes. Run
`docker compose restart searxng` for changes to take effect.

## Planck Docker image

Built from this repository. Includes:

- Elixir/OTP/Mix
- Node.js (`@mozilla/readability` for `web_fetch`)
- The planck binary
- The bundled sidecar (written to workspace on first install only)

Runs as UID/GID 1000. Version is read from the root `VERSION` file.

The sidecar defines a `setup` alias in `mix.exs` that runs `mix deps.get` and
`npm install --prefix priv`. Planck's `SidecarManager` calls `mix setup` on
startup if the alias exists, falling back to `mix deps.get` otherwise. Sidecars
that don't need extra steps don't need to define the alias.

## Bundled sidecar

A Mix project pre-installed in the planck image. Written to
`$HOME/planck/workspace/.planck/sidecar/` on first install only. Preserved
on subsequent starts — the user can customise it freely. All tools use
`erlexec` for process execution.

### `search_workspace`

Queries Typesense at `http://typesense:8108`. Returns ranked results from
indexed workspace files. Retries with backoff if Typesense is not yet ready.

### `search_web`

Queries Searxng at `http://searxng:8080`. Returns web search results.
Retries with backoff if Searxng is not yet ready.

### `web_fetch`

Fetches a URL and strips noise using `@mozilla/readability` via a Node.js
script invoked through `erlexec`. Returns clean markdown.

```elixir
:exec.run(["node", "/app/scripts/readability.js", url],
  [:sync, :stdout, :stderr, {:timeout, 15_000}])
```

### Workspace watcher

A GenServer that runs on sidecar start. On startup it performs a full index
of all existing files in `/workspace/` before switching to watch mode.
Uses `file_system` to watch `/workspace/` for subsequent changes and triggers
incremental Typesense indexing. Retries Typesense connection with backoff.
Indexed types: `.md`, `.txt`, `.ex`, `.exs`, `.js`, `.ts`, `.json`, `.yaml`,
`.toml`, and common code extensions. Excludes `.planck/` system directories
(sessions, sidecar).

## Bundled skills

### `configure-searxng`

Tells agents where `settings.yml` lives inside the container and documents
the key settings (engines, language, safe search, result count). Agents use
`read`/`edit` to apply changes. Always reminds the user that
**`docker compose restart`** is required for changes to take effect.

## Default config

No default config is written. On first launch Planck shows a SetupModal
where the user selects a provider and enters their API key. The resulting
`config.json` is then written to the workspace by the UI.

## Install script

`docs/install_docker.sh` — follows the same structure as `docs/install.sh`:

```sh
curl -fsSL https://thebroken.link/planck/install_docker.sh | sh
```

Options:

| Flag | Default | Description |
|---|---|---|
| `--bind` | `127.0.0.1` | Address to bind planck on the host |

Examples:

```bash
# Local only (default — safe for a non-technical user's machine)
curl -fsSL .../install_docker.sh | sh

# Expose on network (server deployment)
curl -fsSL .../install_docker.sh | sh -s -- --bind 0.0.0.0
```

The script is idempotent — all write steps use an `add_if_missing` pattern so
existing files and `.env` keys are never overwritten. The script:

1. Checks Docker is installed and running; exits with install instructions if not
2. Pulls all Docker images
3. Writes `$HOME/planck/.env` with `PLANCK_BIND_ADDRESS` and `TYPESENSE_API_KEY` (skips keys already present)
4. Copies bundled sidecar to workspace (skips if present)
5. Starts all services: `docker compose up -d`

## Updates

```sh
curl -fsSL https://thebroken.link/planck/install_docker.sh | sh
```

Re-running the install script is safe — it skips steps already completed
and pulls updated Docker images.
