# planck_docker

A Docker Compose stack that runs Planck with a local LLM, web search, and
workspace indexing. Designed to be installed once by a technical user for
themselves or for someone else. The same setup covers both local-only access
(default) and network-accessible server deployments.

## Use cases

- **Local workstation** ŌĆö install on your own machine; access at `localhost:4000`
- **Someone else's machine** ŌĆö install for a non-technical user; they open a browser, nothing else to configure
- **Home server** ŌĆö expose on the local network so other household members can access it from their own devices

## Stack

```
Docker Compose
Ōö£ŌöĆŌöĆ planck          (Elixir web UI + agent runtime + sidecar, Node.js included)
Ōö£ŌöĆŌöĆ llama-cpp       (PrismML-Eng/llama.cpp, CPU)
Ōö£ŌöĆŌöĆ searxng         (self-hosted web search)
ŌööŌöĆŌöĆ typesense       (workspace indexing)
```

All services communicate on an internal Docker network. Only `planck:4000`
is optionally exposed on the host ŌĆö all other services are internal. All
containers run as UID/GID 1000 so files written to the workspace are owned
by the user.

## Network access

Controlled by `PLANCK_BIND_ADDRESS` in `$HOME/planck/.env`:

| Value | Use |
|---|---|
| `127.0.0.1` (default) | Local only ŌĆö no ports open to the network |
| `0.0.0.0` | All interfaces ŌĆö accessible from other devices |

```yaml
# compose.yml
ports:
  - "${PLANCK_BIND_ADDRESS:-127.0.0.1}:4000:4000"
```

## Directory layout

```
$HOME/planck/
  .env                        ŌåÉ generated on first install; edit to configure
  models/
    Bonsai-8B-Q1_0.gguf       (1.16 GB, downloaded on first install)
  workspace/                  ŌåÉ mounted into the planck container as cwd
    .planck/
      config.json             ŌåÉ written on first install only; preserved afterwards
      sidecar/                ŌåÉ written on first install only; preserved afterwards
      sessions/
      skills/                 ŌåÉ user-managed
      teams/                  ŌåÉ user-managed
```

`~/.planck/` (the global Planck CLI config) is never touched.
The planck container's cwd is `$HOME/planck/workspace/`, so it picks up
`.planck/config.json` from there and never reads from `~/.planck/`.

## `.env` file

Generated by `install_docker.sh` on first install:

```bash
# $HOME/planck/.env
TYPESENSE_API_KEY=planck-internal-key
PLANCK_BIND_ADDRESS=127.0.0.1
```

Edit `PLANCK_BIND_ADDRESS` to `0.0.0.0` to expose on the network.
Docker Compose reads this file automatically.

## Bonsai model

- **File:** `Bonsai-8B-Q1_0.gguf` ŌĆö 1.16 GB
- **Download URL:** `https://huggingface.co/prism-ml/Bonsai-8B-gguf/resolve/main/Bonsai-8B-Q1_0.gguf`
- **Context:** 65,536 tokens
- **API:** OpenAI-compatible at `http://llama-cpp:11434/v1` (internal only)
- **Server invocation:**
  ```
  llama-server -m /models/Bonsai-8B-Q1_0.gguf --host 0.0.0.0 --port 11434
  ```

## Internal service credentials

Typesense requires an API key. Since it is internal to the Docker network
and never bound to the host, a fixed key in `.env` is safe:

```yaml
services:
  typesense:
    environment:
      TYPESENSE_API_KEY: ${TYPESENSE_API_KEY}
  planck:
    environment:
      TYPESENSE_API_KEY: ${TYPESENSE_API_KEY}
```

## Searxng configuration

A sensible default `settings.yml` is bundled and mounted into the searxng
container. It enables major search engines, sets a neutral locale, and
disables tracking features.

Users can customise Searxng through Planck via the `configure-searxng` skill.
The skill documents the settings file location and key options; agents use
`read`/`edit` to apply changes. **planck_docker must be restarted** for
Searxng configuration changes to take effect (`docker compose restart`).

## Planck Docker image

Built from this repository. Includes:

- Elixir/OTP/Mix
- Node.js (`@mozilla/readability` for `web_fetch`)
- The planck binary
- The bundled sidecar (written to workspace on first install only)

Runs as UID/GID 1000.

## Bundled sidecar

A Mix project pre-installed in the planck image. Written to
`$HOME/planck/workspace/.planck/sidecar/` on first install only. Preserved
on subsequent starts ŌĆö the user can customise it freely. All tools use
`erlexec` for process execution.

### `search_workspace`

Queries Typesense at `http://typesense:8108`. Returns ranked results from
indexed workspace files. Retries with backoff if Typesense is not yet ready.

### `search_web`

Queries Searxng at `http://searxng:8080`. Returns web search results.
Retries with backoff if Searxng is not yet ready.

### `web_fetch`

Fetches a URL and strips noise using `@mozilla/readability` via a Node.js
script invoked through `erlexec`. Returns clean markdown.

```elixir
:exec.run(["node", "/app/scripts/readability.js", url],
  [:sync, :stdout, :stderr, {:timeout, 15_000}])
```

### Workspace watcher

A GenServer that runs on sidecar start. On startup it performs a full index
of all existing files in `/workspace/` before switching to watch mode.
Uses `file_system` to watch `/workspace/` for subsequent changes and triggers
incremental Typesense indexing. Retries Typesense connection with backoff.
Indexed types: `.md`, `.txt`, `.ex`, `.exs`, `.js`, `.ts`, `.json`, `.yaml`,
`.toml`, and common code extensions. Excludes `.planck/` system directories
(sessions, sidecar).

## Bundled skills

### `configure-searxng`

Tells agents where `settings.yml` lives inside the container and documents
the key settings (engines, language, safe search, result count). Agents use
`read`/`edit` to apply changes. Always reminds the user that
**`docker compose restart`** is required for changes to take effect.

## Install script

`docs/install_docker.sh` ŌĆö follows the same structure as `docs/install.sh`:

```sh
curl -fsSL https://alexdesousa.github.io/planck/install_docker.sh | sh
```

Options:

| Flag | Default | Description |
|---|---|---|
| `--bind` | `127.0.0.1` | Address to bind planck on the host |

Examples:

```bash
# Local only (default ŌĆö safe for a non-technical user's machine)
curl -fsSL .../install_docker.sh | sh

# Expose on network (server deployment)
curl -fsSL .../install_docker.sh | sh -s -- --bind 0.0.0.0
```

The script:
1. Checks Docker is installed and running; exits with install instructions if not
2. Downloads `Bonsai-8B-Q1_0.gguf` to `$HOME/planck/models/` (skips if present)
3. Pulls all Docker images
4. Writes `$HOME/planck/.env` with `PLANCK_BIND_ADDRESS` and `TYPESENSE_API_KEY` (skips if present)
5. Writes default `config.json` and copies bundled sidecar to workspace (skips if present)
6. Starts all services: `docker compose up -d`

## Updates

```sh
curl -fsSL https://alexdesousa.github.io/planck/install_docker.sh | sh
```

Re-running the install script is safe ŌĆö it skips steps already completed
(model, config, sidecar) and pulls updated Docker images.
